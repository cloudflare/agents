# Memory Provider Interfaces

> **Status**: Draft
> **Date**: 2025-02-25

## Overview

One provider interface per memory type. Providers handle storage — the memory layer handles scoping, validation, and orchestration.

## Session Provider

Stores conversation messages and compaction summaries.

```typescript
interface SessionProvider {
  // Messages
  getMessages(sessionId: string, options?: MessageQueryOptions): Promise<StoredMessage[]>;
  appendMessages(sessionId: string, messages: StoredMessage[]): Promise<void>;
  deleteMessages(sessionId: string, messageIds: string[]): Promise<void>;
  clearMessages(sessionId: string): Promise<void>;
  countMessages(sessionId: string): Promise<number>;

  // Summaries (from compaction)
  getSummaries(sessionId: string): Promise<StoredSummary[]>;
  saveSummary(sessionId: string, summary: StoredSummary): Promise<void>;

  // Session lifecycle
  deleteSession(sessionId: string): Promise<void>;
  listSessions(options?: ListOptions): Promise<string[]>;
}

interface MessageQueryOptions {
  limit?: number;
  offset?: number;
  before?: string;  // ISO timestamp
  after?: string;   // ISO timestamp
  role?: string;
}

interface StoredMessage {
  id: string;
  role: string;
  content: string;
  createdAt: string;  // ISO timestamp
  metadata?: Record<string, unknown>;
}

interface StoredSummary {
  id: string;
  content: string;
  createdAt: string;
  compactedMessageIds: string[];
  tokensBefore: number;
  tokensAfter: number;
}
```

## Working Memory Provider

Stores blocks that go into the prompt.

```typescript
interface WorkingMemoryProvider {
  getBlock(label: string): Promise<StoredBlock | null>;
  getAllBlocks(): Promise<StoredBlock[]>;
  setBlock(label: string, block: StoredBlock): Promise<void>;
  deleteBlock(label: string): Promise<void>;
  clear(): Promise<void>;
}

interface StoredBlock {
  label: string;
  content: string;
  limit: number;
  description?: string;
  updatedAt: string;  // ISO timestamp
}
```

## Archival Provider

Stores passages with embeddings. Includes search.

```typescript
interface ArchivalProvider {
  // Storage
  get(id: string): Promise<StoredPassage | null>;
  put(passage: StoredPassage): Promise<void>;
  putBatch(passages: StoredPassage[]): Promise<void>;
  delete(id: string): Promise<void>;
  deleteBatch(ids: string[]): Promise<void>;
  list(options?: ListOptions): Promise<StoredPassage[]>;
  count(options?: FilterOptions): Promise<number>;

  // Search
  search(query: SearchQuery): Promise<SearchResult[]>;

  // Embeddings (provider may generate or expect them)
  readonly generateEmbeddings: boolean;
}

interface StoredPassage {
  id: string;
  text: string;
  embedding?: number[];  // may be generated by provider
  source?: string;
  tags?: string[];
  metadata?: Record<string, unknown>;
  createdAt: string;
}

interface SearchQuery {
  text: string;
  embedding?: number[];  // pre-computed, or provider generates
  topK?: number;
  minScore?: number;
  filter?: {
    source?: string;
    tags?: string[];
    before?: string;
    after?: string;
  };
}

interface SearchResult {
  id: string;
  score: number;
  passage: StoredPassage;
}

interface ListOptions {
  limit?: number;
  offset?: number;
  source?: string;
  tags?: string[];
}

interface FilterOptions {
  source?: string;
  tags?: string[];
}
```

## Embedding Handling

Some providers generate embeddings, some expect them:

```typescript
// Provider generates embeddings (e.g., AISearch)
const provider = new AISearchProvider(env.AI_SEARCH);
provider.generateEmbeddings; // true
await provider.put({ id: '1', text: 'hello' }); // embedding auto-generated
await provider.search({ text: 'hi' }); // embedding auto-generated

// Provider expects embeddings (e.g., Vectorize)
const provider = new VectorizeProvider(env.VECTORIZE);
provider.generateEmbeddings; // false
await provider.put({ id: '1', text: 'hello', embedding: [...] }); // must provide
await provider.search({ text: 'hi', embedding: [...] }); // must provide
```

When `generateEmbeddings: false`, the memory layer handles embedding generation.

## Common Patterns

**1. Timestamps as ISO strings**
```typescript
createdAt: string;  // "2025-02-25T12:00:00.000Z"
```

**2. IDs are strings**
```typescript
id: string;  // Provider doesn't care about format
```

**3. Scoping handled by memory layer**
```typescript
// Provider stores what it's given
// Memory layer prefixes/namespaces as needed
```

**4. Batch operations**
```typescript
putBatch(passages: StoredPassage[]): Promise<void>;
deleteBatch(ids: string[]): Promise<void>;
```
