Add `agents/observability/ai` with `wrapAISDK` for AI SDK v6 and `createAISDKTelemetry` for AI SDK v7. `wrapAISDK` instruments a v6 SDK namespace; `createAISDKTelemetry` returns a v7 `Telemetry` integration for `registerTelemetry` or per-call `experimental_telemetry.integrations`. Both project into the same schema. Instrumented operations produce Cloudflare-native spans using the scalar OpenTelemetry GenAI attributes supported by Workers: `invoke_agent {agent}`, `chat {model}`, and `execute_tool {tool}` spans with request settings, token usage, dashboard-ready total/tool counts, scalar finish reasons, tool-call IDs, and model-call time to first chunk. By default these spans carry no conversation content — prompts, messages, tool inputs/outputs, and raw model output are omitted, and schemas, headers, provider options, and raw error messages are never recorded. You can opt in to recording chat and tool inputs/outputs (potentially PII) with `recordInputs`/`recordOutputs`, mirroring the AI SDK's own `TelemetrySettings`: set them on the `wrapAISDK` options or per call via `experimental_telemetry` for v6, or on `createAISDKTelemetry(options)` for v7. Both default to `false`; recorded content is JSON-serialized and truncated to a safe byte cap. The underlying Workers tracing adapter remains internal.
