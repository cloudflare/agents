# 24 — End-to-end scenarios

A handful of full-composition tests in `src/e2e/`, using only memory adapters,
FakeModel, and the public `Agent`/`Think` APIs. These are the acceptance bar
for "high-level functionality ported". Each is one test file.

## 1. `chat-session.e2e.test.ts` — the everyday chat
- Think subclass with a system prompt, one user tool (`add`), workspace on.
- Connect a MemoryConnection; send `cf_agent_use_chat_request` with a user
  message; FakeModel script: tool-call(add) → text answer.
- Assert: chunk frames stream in order; tool executed with validated input;
  final assistant message persisted with tool part + text; a second
  connection receives full history on connect; `message:response` event.

## 2. `durable-work.e2e.test.ts` — schedules, queue, fibers surviving "eviction"
- Agent subclass with a scheduled method (cron + interval), a queued task,
  and a startFiber with stash.
- Simulate eviction: recreate the agent over the same KeyValueStore/Alarm
  state (fresh instances, no in-memory carryover), advance TestClock.
- Assert: schedules fire on time and recur; queue drains FIFO with retry;
  interrupted fiber recovers via onFiberRecovered with the last stash;
  managed fiber duplicate-start dedupes by idempotency key.

## 3. `submissions-scheduled-tasks.e2e.test.ts` — programmatic turns
- Think with `getScheduledTasks()` (a prompt task on `every day at 09:00 in
  UTC` — DSL exercised) and external `submitMessages` calls.
- Assert: accept-before-run; FIFO isolation (second submission's messages
  invisible until its turn); idempotent retry; cancellation of a pending
  submission; scheduled occurrence creates a deduped submission and re-arms
  the next occurrence; late alarm does not backfill.

## 4. `recovery.e2e.test.ts` — interruption + stall + overflow
- Think with chatRecovery `{ maxAttempts: 2, terminalMessage }`,
  stall timeout, contextOverflow reactive + classifier.
- Scenarios: (a) FakeModel emits partial text then hangs → watchdog stall →
  continuation turn completes the message; (b) always-hanging model →
  exhaustion → terminal message persisted + `chat:recovery:exhausted`;
  (c) overflow-classified error → compaction (session with compactAfter +
  fake summarizer) → retry succeeds.

## 5. `actions-approvals.e2e.test.ts` — the HITL path
- Think with actions: idempotent `charge` (explicit key), approval-gated
  `delete_account`, durable-pause `deploy` with permissions + narrowed
  `authorizeTurn`.
- Assert: ledger replay on recovery-style duplicate call; approval frame
  approve/reject; durable-pause parks → turn ends → `approveExecution` runs
  once, writes output into the message, auto-continues; unauthorized action
  returns ActionAuthorizationError value to the model; attachments surface in
  `onChatResponse`.

## 6. `delegation.e2e.test.ts` — parent/child agent tool
- Parent Think with `agentTool(ResearcherThink)`; MemoryAgentSpawner registered
  with both classes.
- FakeModel (parent): tool-call(research) → final text using child summary.
  FakeModel (child): text answer.
- Assert: run registry lifecycle; child events tailable/replayable; cancel
  path; parent "eviction" mid-run → reconcile settles from live child.

## Conventions
- No sleeps: TestClock + injected timers drive everything.
- Each scenario asserts observability events as a secondary signal (the event
  stream is part of the ported surface).
- Keep each file < ~300 lines; these are smoke-level, not exhaustive — module
  tests own the edge cases.
